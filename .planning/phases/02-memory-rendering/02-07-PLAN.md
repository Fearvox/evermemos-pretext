# Plan 02-07: Cursor-Based Pagination, Search Integration, and End-to-End Wiring

<objective>
Wire cursor-based infinite scroll pagination into the virtualized list via useInfiniteQuery, integrate search highlighting with Pretext measurement, and verify the full data flow end-to-end: mock API → useInfiniteQuery → VirtualizedMemoryList → Pretext heights → smooth scrolling of 1000+ items.
</objective>

<must_haves>
- useInfiniteQuery fetches pages with cursor-based pagination (50 items/page)
- Infinite scroll triggers fetchNextPage when last visible item is within 5 of total count
- Filter/search changes reset to cursor=null (first page) via queryKey change
- Search highlighting wraps matched terms in <mark> without affecting Pretext measurement
- 1000+ mock items scroll without visible scroll jumps
- PaginationIndicator shows "Loading more..." or "You've reached the end"
</must_haves>

<action>
Update `VirtualizedMemoryList.tsx` to trigger pagination:

1. Add a `useEffect` that watches the virtualizer's virtual items:
   ```tsx
   const virtualItems = virtualizer.getVirtualItems()
   const lastItem = virtualItems[virtualItems.length - 1]

   useEffect(() => {
     if (!lastItem) return
     if (lastItem.index >= items.length - 5 && hasMore && !isLoadingMore) {
       onLoadMore?.()
     }
   }, [lastItem?.index, items.length, hasMore, isLoadingMore, onLoadMore])
   ```

2. After the virtual items render area, add:
   ```tsx
   {(hasMore || isLoadingMore) && (
     <PaginationIndicator hasMore={hasMore} isLoading={isLoadingMore} />
   )}
   ```

3. When `items` array grows (new page loaded), the virtualizer automatically includes new items because `count` updates.

4. On search/filter change (items array resets to new data): scroll to top:
   ```tsx
   const prevQueryRef = useRef(searchQuery)
   useEffect(() => {
     if (prevQueryRef.current !== searchQuery) {
       virtualizer.scrollToIndex(0)
       prevQueryRef.current = searchQuery
     }
   }, [searchQuery, virtualizer])
   ```
</action>
