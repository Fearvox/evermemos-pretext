# Plan 1: Next.js Hub Scaffold + Submodule Infrastructure

<objective>
Create the Next.js 16 App Router hub at `hub/`, wire Pretext as a git submodule dependency via path alias + transpilePackages, establish dual package manager isolation (npm for hub, bun for pretext), set up the TypeScript compilation boundary so hub's tsc never picks up Pretext's TS 6 flags, define the PRETEXT_FONT constant as the single source of truth for the Inter font string, and configure the Vercel build command so end-to-end CI works from a clean clone.
</objective>

<must_haves>
- `hub/` directory exists with a working Next.js 16 App Router project
- `npm run dev` in `hub/` starts on port 3001 without errors
- `pretext/` submodule is at the repo root (or already present) and `dist/layout.js` is generated by `bash scripts/setup.sh`
- `@pretext` path alias resolves to `../pretext/src/layout.ts` in both Turbopack and webpack
- Hub's `tsc --noEmit` succeeds and does NOT include `../pretext/src/` in compilation
- `PRETEXT_FONT = '16px Inter'` constant exists in `hub/src/lib/pretext/fonts.ts`
- Inter is loaded via `next/font/google` in root layout with `variable: '--font-inter'`
- `scripts/setup.sh` chains submodule init + bun build:package + npm install
- `vercel.json` has `buildCommand`, `installCommand`, `outputDirectory` for Vercel CI
</must_haves>

<task id="1.2">
<title>Configure next.config.ts for Pretext submodule consumption</title>
<read_first>
- .planning/phases/01-foundation/01-RESEARCH.md (next.config.ts section, transpilePackages + resolveAlias rationale)
- .planning/phases/01-foundation/01-CONTEXT.md (D-01: path alias + transpilePackages + Turbopack resolveAlias)

<action>
Create or replace `hub/next.config.ts` with the following content. This wires the `@pretext` alias to the submodule source for both Turbopack (default bundler in Next.js 16) and webpack (fallback):

```ts
// hub/next.config.ts
import type { NextConfig } from 'next'
import path from 'path'

const nextConfig: NextConfig = {
  transpilePackages: ['@chenglou/pretext'],
  experimental: {
    turbo: {
      resolveAlias: {
        '@pretext': path.resolve(__dirname, '../pretext/src/layout.ts'),
      },
    },
  },
  webpack(config) {
    config.resolve.alias['@pretext'] = path.resolve(__dirname, '../pretext/src/layout.ts')
    return config
  },
}

export default nextConfig
```

Note: `__dirname` in `next.config.ts` resolves to `hub/`, so `../pretext/src/layout.ts` correctly targets the submodule root.
</action>

<action>
Update `hub/tsconfig.json` to add the `@pretext` path alias and explicitly exclude `../pretext/src` from compilation. The key additions (merge into whatever create-next-app generated):

```json
{
  "compilerOptions": {
    "target": "ES2017",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "esnext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "plugins": [
      {
        "name": "next"
      }
    ],
    "paths": {
      "@/*": ["./src/*"],
      "@pretext": ["../pretext/src/layout.ts"]
    }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules", "../pretext/src", "../pretext/src/**"]
}
```

The `exclude` entries for `../pretext/src` prevent hub's TypeScript 5.x compiler from picking up TS 6-specific flags (`allowImportingTsExtensions`, `verbatimModuleSyntax`) from Pretext's own `tsconfig.json`.
</action>

<action>
Step 1 — Update `hub/src/app/layout.tsx` to load Inter via next/font/google:

```tsx
// hub/src/app/layout.tsx
import type { Metadata } from 'next'
import { Inter } from 'next/font/google'
import './globals.css'

const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
  variable: '--font-inter',
})

export const metadata: Metadata = {
  title: 'EverMemOS Hub',
  description: 'Memory content powered by Pretext text layout',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={inter.variable}>
      <body className={inter.className}>
        {children}
      </body>
    </html>
  )
}
```

Step 2 — Create `hub/src/lib/pretext/fonts.ts` as the single source of truth for the Pretext font string:

```ts
// hub/src/lib/pretext/fonts.ts
// CLIENT ONLY — do not import from Server Components
//
// Single source of truth for the Pretext font string.
// Must exactly match the CSS font applied to text blocks.
//
// Rules:
//   - Must be a CSS font shorthand: '<size>px <family>'
//   - Size must match the actual rendered font size in DOM (default body: 16px)
//   - Family must match what the browser resolved — 'Inter' (capital I, next/font/google)
//   - NEVER use system-ui, -apple-system, or CSS variables here.
//     Canvas and DOM resolve system-ui to different optical variants on macOS.
export const PRETEXT_FONT = '16px Inter'

// Line height used with layout() calls. Must match the CSS line-height of text blocks.
// 1.5 * 16px = 24px
export const PRETEXT_LINE_HEIGHT = 24
```

Step 3 — In `hub/src/app/globals.css`, add a utility class that matches the Pretext font exactly so both CSS and canvas use identical font properties:

```css
/* hub/src/app/globals.css */
@import "tailwindcss";

/* Pretext text block — font must match PRETEXT_FONT constant exactly */
.pretext-block {
  font-family: 'Inter', sans-serif;
  font-size: 16px;
  line-height: 24px;
}
```
</action>

<title>Create scripts/setup.sh and vercel.json for Vercel CI</title>

- .planning/phases/01-foundation/01-CONTEXT.md (D-13: setup.sh chain, D-14: Vercel build command)
</read_first>
<action>
Step 1 — Create `scripts/setup.sh` at the repo root:

```bash
#!/usr/bin/env bash
set -e

echo "=== EverMemOS-Pretext Setup ==="

echo "[1/3] Initializing submodules..."
git submodule update --init --recursive

echo "[2/3] Building Pretext package (bun)..."
cd pretext
bun install
bun run build:package
cd ..

echo "[3/3] Installing Hub dependencies (npm)..."
cd hub
npm install
cd ..

echo "=== Setup complete. ==="
echo "  Pretext dist: pretext/dist/layout.js"
echo "  Hub dev:      cd hub && npm run dev  (port 3001)"
```

Make it executable:
```bash
chmod +x scripts/setup.sh
```

Step 2 — Create `vercel.json` at the repo root:

```json
{
  "buildCommand": "bash scripts/setup.sh && cd hub && npm run build",
  "installCommand": "echo 'Install handled by buildCommand'",
  "outputDirectory": "hub/.next",
  "framework": "nextjs"
}
```

Note: Vercel needs the repo root to contain `vercel.json`. The `buildCommand` explicitly runs submodule init before the next build. If Vercel's project settings have `rootDirectory` set to `hub/`, adjust `buildCommand` to `bash ../scripts/setup.sh` instead — but using the repo root with explicit paths is more robust.

Step 3 — Verify `scripts/` directory does not already contain conflicting setup scripts:
```bash
ls scripts/
```
If `scripts/police/` or other scripts exist, they can coexist — do not delete them.
</action>
